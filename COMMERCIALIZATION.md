# Commercialization Boundary

Current public repositories are released as open source unless a specific file says otherwise.

The reusable Control Layer OS template should stay broadly adoptable. Future commercial work should be separated into clearly scoped products, hosted services, private integrations, support agreements, or proprietary modules instead of making the public profile or template ambiguous.

Apache-2.0 covers source and documentation. It does not grant permission to use Sirius branding in a way that implies endorsement, ownership, or official status for unrelated services.
